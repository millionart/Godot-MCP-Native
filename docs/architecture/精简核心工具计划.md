## 3. 修正后的方案：精简到 30 个 core 工具

目标：46 → 30，需净移出 16 个（移出 18 个到 supplementary，移入 2 个从 supplementary 到 core）。

### 3.1 从 Core 移出到 Supplementary（18 个）

**移到 Node-Write-Advanced（4 个）：**
| 工具 | 移出理由 |
|------|---------|
| set_anchor_preset | UI 布局专用，非日常节点操作 |
| connect_signal | 信号连接是高级操作，AI 较少主动调用 |
| disconnect_signal | 信号断开，同上 |
| set_node_groups | 节点组设置，使用频率低 |

**移到 Script-Advanced（3 个）：**
| 工具 | 移出理由 |
|------|---------|
| analyze_script | 静态分析，非编码日常操作 |
| validate_script | 验证用途单一，不常用 |
| search_in_files | 编辑器自带搜索功能可替代 |

**移到 Scene-Advanced（1 个）：**
| 工具 | 移出理由 |
|------|---------|
| list_project_scenes | 列出场景列表，低频操作 |

**移到 Editor-Advanced（4 个）：**
| 工具 | 移出理由 |
|------|---------|
| set_editor_setting | 编辑器设置修改，低频且风险较高 |
| get_editor_screenshot | 截图功能，特定场景才需要 |
| get_signals | 获取信号列表，低频查询操作 |
| get_selected_nodes | 选择状态查询，AI 更常通过 get_editor_state 获取 |

**移到 Project-Advanced（2 个）：**
| 工具 | 移出理由 |
|------|---------|
| create_resource | 创建资源文件，比读取低频 |
| get_project_structure | 项目结构树，get_project_info 已含基本信息 |

**额外移出（4 个，原计划遗漏）：**
| 工具 | 当前组 | 移到 | 移出理由 |
|------|--------|------|---------|
| add_resource | Node-Write | Node-Write-Advanced | 与 create_resource 语义重叠，项目资源创建比节点操作低频 |
| get_node_groups | Node-Read | Node-Advanced | 节点组查询，低频操作 |
| find_nodes_in_group | Node-Read | Node-Advanced | 按组查找，低频操作 |
| get_scene_structure | Scene | Scene-Advanced | 场景结构树查询，get_scene_tree 已提供树信息 |

### 3.2 从 Supplementary 移入 Core（2 个）

| 工具 | 当前位置 | 移到 | 移入理由 |
|------|---------|------|---------|
| execute_script | Debug-Advanced | Script | AI 最常用的脚本执行入口，应为核心 |
| execute_editor_script | Editor-Advanced | Editor | 编辑器脚本执行，AI 高频使用 |

### 3.3 调整后的 Core 工具分布（30 个）

**Node-Write（6 个）：** create_node, delete_node, update_node_property, duplicate_node, move_node, rename_node

**Node-Read（3 个）：** get_node_properties, list_nodes, get_scene_tree

**Script（7 个）：** list_project_scripts, read_script, create_script, modify_script, get_current_script, attach_script, **execute_script**

**Scene（4 个）：** create_scene, save_scene, open_scene, get_current_scene

**Editor（4 个）：** get_editor_state, run_project, stop_project, **execute_editor_script**

**Debug（3 个）：** get_editor_logs, debug_print, clear_output

**Project（3 个）：** get_project_info, get_project_settings, list_project_resources

**合计：6+3+7+4+4+3+3 = 30** ✅

## 4. 分类合理性校验

### 4.1 移出合理性

| 工具 | 核心判断标准 | 结论 |
|------|------------|------|
| set_anchor_preset | AI 编码时极少直接设锚点，通常通过 modify_script 写布局代码 | ✅ 移出 |
| connect_signal / disconnect_signal | 信号连接属于"胶水代码"，AI 更常通过 modify_script 在 _ready 中写代码 | ✅ 移出 |
| set_node_groups | 节点组属组织性操作，非功能性操作 | ✅ 移出 |
| analyze_script / validate_script | 诊断类工具，非编码日常 | ✅ 移出 |
| search_in_files | 编辑器 Ctrl+Shift+F 可替代，AI 也可用 read_script + 自行搜索 | ✅ 移出 |
| list_project_scenes | 低频列表查询 | ✅ 移出 |
| set_editor_setting | 修改编辑器配置有风险，不应默认暴露 | ✅ 移出 |
| get_editor_screenshot | 视觉调试专用 | ✅ 移出 |
| get_signals | 信号元数据查询，低频 | ✅ 移出 |
| get_selected_nodes | get_editor_state 已含选择信息 | ⚠️ 可议，但移出合理 |
| create_resource | 资源创建低频且需明确意图 | ✅ 移出 |
| get_project_structure | 与 get_project_info 信息重叠 | ✅ 移出 |
| add_resource | 与 create_resource 重叠，且放 Node-Write 语义不当 | ✅ 移出 |
| get_node_groups / find_nodes_in_group | 节点组查询低频 | ✅ 移出 |
| get_scene_structure | get_scene_tree 已提供树形结构 | ✅ 移出 |

### 4.2 移入合理性

| 工具 | 核心判断标准 | 结论 |
|------|------------|------|
| execute_script | AI 执行代码片段最直接的方式，高频刚需 | ✅ 移入 |
| execute_editor_script | 编辑器上下文执行，高频刚需 | ✅ 移入 |

### 4.3 潜在争议项

| 工具 | 争议 | 建议 |
|------|------|------|
| get_selected_nodes | 移出后 AI 需额外启用 Editor-Advanced 组 | 保留移出，get_editor_state 足够 |
| add_resource | 移到 Node-Write-Advanced 语义仍不完美 | 可考虑移到 Project-Advanced（资源创建更偏项目操作） |
